Earlier this week [Dub.co](http://Dub.co) shared our [customer story](https://dub.co/customers/tamias) on why we choose Dub as our link-sharing infrastructure.

<br />
In this blog post where gonna share a little more in details how we implemented this
functionality.

<br />
We have some features like Time Tracker, Reports and files from Vault that our users
share outside their company. Because the authorization layer still lives on our
side, these links can get really long because they include a unique token.
<br />
Our solution was to implement Dub to generate unique short URLs.

<br />
### How we implemented sharing for our reports

<br />
![Tamias - Overview](/images/overview.png)

<br />
If you look closely you can se our link looks like this: [https://go.tamias.xyz/5eYKrmV](https://go.tamias.xyz/5eYKrmV)

<br />
When the user clicks `Share` we execute a typed server action

that looks like this:

```typescript
const createReport = useAction(createReportAction, {
  onError: () => {
    toast({
      duration: 2500,
      variant: "error",
      title: "Something went wrong pleaase try again.",
    });
  },
  onSuccess: (data) => {
    setOpen(false);

    const { id } = toast({
      title: "Report published",
      description: "Your report is ready to share.",
      variant: "success",
      footer: (
        <div className="mt-4 space-x-2 flex w-full">
          <CopyInput
            value={data.short_link}
            className="border-[#2C2C2C] w-full"
          />

          <Link href={data.short_link} onClick={() => dismiss(id)}>
            <Button>View</Button>
          </Link>
        </div>
      ),
    });
  },
});
```

<br />

The nice thing with this approach is that you get callbacks on `onError` and `onSuccess` so in this case we show a toast based on the callback.

<br />

The action is pretty straightforward too, we first save the report based on the current parameters (from, to and type) depending on what kind of report we are creating.

<br />
We save it in our database and get an id back that we use to generate our shareable URL.

```typescript
const dub = new Dub({ projectSlug: "tamias" });

export const createReportAction = action(schema, async (params) => {
  const db = await getDatabase();
  const user = await getUser();

  const report = await createReport({
    db,
    teamId: user.data.team_id,
    from: params.from,
    to: params.to,
    type: params.type,
    expiresAt: params.expiresAt,
  });

  const link = await dub.links.create({
    url: `${params.baseUrl}/report/${report.id}`,
    rewrite: true,
    expiresAt: params.expiresAt,
  });

  const publishedReport = await updateReportShareLink({
    db,
    reportId: report.id,
    linkId: link.id,
    shortLink: link.shortLink,
  });

  const logsnag = await setupLogSnag();

  logsnag.track({
    event: LogEvents.OverviewReport.name,
    icon: LogEvents.OverviewReport.icon,
    channel: LogEvents.OverviewReport.channel,
  });

  return publishedReport;
});
```

<br />
With the combination of server actions, our database layer and Dub we can create really beautiful
URLs with analytics on top.
