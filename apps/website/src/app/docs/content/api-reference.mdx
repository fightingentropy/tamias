The Tamias API lets you access your financial data programmatically. Build custom integrations, automate workflows, or connect Tamias to your own tools.

## Full API documentation

For complete API documentation including all endpoints, parameters, and examples:

**[api.tamias.xyz](https://api.tamias.xyz)**

## Overview

The API provides access to:

- Transactions
- Invoices
- Customers
- Time tracking projects and entries
- Financial reports
- Documents
- Bank accounts

## Authentication

The Tamias API supports two authentication methods:

### API Keys (for your own data)

For accessing your own Tamias data in scripts, automations, or personal tools.

1. Go to [Settings → Developer](https://tamias.xyz/settings/developer)
2. Click **Create API key**
3. Give it a descriptive name
4. Copy the key immediately (you won't see it again)

Include your API key in the `Authorization` header:

```
Authorization: Bearer your-api-key-here
```

### OAuth 2.0 (for user-facing apps)

For building apps that access other users' Tamias data with their permission.

1. Create an OAuth application in [Settings → Developer](https://tamias.xyz/settings/developer)
2. Implement the OAuth authorization flow
3. Exchange authorization codes for access tokens
4. Use access tokens in API requests

[Build an OAuth app →](/docs/build-oauth-app)

## Base URL

All API endpoints use:

```
https://api.tamias.xyz/v1
```

## Rate limits

- **Standard**: 100 requests per minute
- **Burst**: Up to 200 requests in short bursts

If you exceed rate limits, you'll receive a `429 Too Many Requests` response. Wait and retry with exponential backoff.

## Access methods

### HTTPS API

Use standard HTTPS requests with your bearer token:

```bash
curl https://api.tamias.xyz/v1/transactions \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### MCP package

For AI tool integrations:

```bash
npx @tamias/mcp
```

[MCP documentation →](/docs/assistant-mcp)
